# Design Guide — Navigation

> Claude가 UI 디자인 작업 시 참조하는 문서.
> 서비스 컨텍스트, 디자인 원칙, 화면 명세, 컴포넌트 규칙을 담는다.

---

## 1. 서비스 한 줄 정의

> "비콘도, Wi-Fi 측위맵도 없이 — 스마트폰 센서와 건물 평면도 한 장만으로 실내에서 길을 찾는다."

경진대회 제출용 Flutter 모바일 앱. 실외(GPS) → 실내(PDR + Particle Filter) 자동 전환이 핵심 시연 포인트이며, 자연어("3층 카페 어디야?") 목적지 입력을 RAG로 지원한다.

---

## 2. 사용자 프로파일

| 구분 | 설명 |
|---|---|
| 1차 타겟 | 쇼핑몰·병원·공항 등 복잡한 실내를 처음 방문한 일반인 |
| 기술 수준 | 지도 앱(네이버·카카오) 사용에 익숙한 비전문가 |
| 사용 상황 | 이동 중 한 손으로 빠르게 목적지 확인; 시선은 지도에 짧게만 둔다 |
| 데모 대상 | 경진대회 심사위원 — "와, 신기하다"는 임팩트가 중요 |

---

## 3. 핵심 사용자 플로우 (데모 골든패스)

```
앱 시작
  └─ [야외 화면] GPS 지도 + 현재 위치 파란 점
       └─ 건물 입구 진입
            └─ 자동 감지 → 전환 애니메이션 (0.5~1초)
                 └─ [실내 화면] 평면도 + 현재 위치 갱신 (실시간)
                      └─ 목적지 입력 (자연어 or 터치)
                           └─ 경로 오버레이 표시
                                └─ 도착
```

이 플로우가 디자인의 중심축. 모든 화면은 이 흐름을 방해하지 않아야 한다.

---

## 4. 디자인 원칙

### 4-1. 지도가 주인공
- UI 크롬(헤더·버튼·패널)은 지도 위 최소 레이어로 올린다.
- 불필요한 텍스트·아이콘은 제거. 지도를 가리는 요소는 최소화.
- 핵심 정보(현재 위치, 경로, 목적지)는 지도 위에서 직접 읽힌다.

### 4-2. 단계별 집중
- 한 화면에 하나의 목표. 실외 → 전환 → 실내 → 경로 단계가 명확히 구분된다.
- 전환 순간은 명확한 시각 피드백(애니메이션 or 토스트)으로 알린다.

### 4-3. 한 손 엄지 도달 범위
- 주요 액션(목적지 입력, 경로 시작)은 화면 하단에 배치.
- FAB, 검색 바, 바텀시트 패턴 우선.

### 4-4. 신뢰 시각화
- 위치 불확실성(Particle Filter 분산)을 투명도 원으로 표현해 "내가 어디 있는지"를 직관적으로 전달.
- 경로는 걸음마다 실시간으로 업데이트 → "살아있는" 느낌.

## 5. 디자인 시스템

### 색상

| 역할 | 값 | 사용처 |
|---|---|---|
| Primary | `#1A73E8` (구글 블루 계열) | 경로 선, 현재 위치 마커, CTA 버튼 |
| Surface | `#FFFFFF` | 바텀시트, 카드 배경 |
| Background | `#F5F5F5` | 스크린 배경 |
| On-surface | `#212121` | 본문 텍스트 |
| Muted | `#757575` | 서브 텍스트, 비활성 상태 |
| Success | `#34A853` | 실내 진입 감지 토스트, 도착 |
| Warning | `#FBBC04` | 위치 정확도 낮음 배지 |
| Error | `#EA4335` | 경로 불가, 센서 오류 |
| Indoor accent | `#6C3FE0` | 실내 전용 마커·경로. 실외(파랑)와 색으로 모드 구분 |

다크모드는 1차 버전 범위 외. 단, 배경·텍스트 contrast는 WCAG AA(4.5:1) 이상 유지.

### 타이포그래피

Flutter 기본 `TextTheme` 기준.

| 역할 | Style | 크기 / Weight |
|---|---|---|
| 건물명·층 레이블 | `titleMedium` | 16sp / 600 |
| 안내 문자 | `bodyLarge` | 16sp / 400 |
| 서브 정보 | `bodySmall` | 12sp / 400 |
| 버튼 | `labelLarge` | 14sp / 600 |

### 간격·레이아웃

- 기본 단위: 8dp
- 바텀시트 상단 패딩: 16dp
- 카드 radius: 16dp
- FAB radius: 16dp (Extended FAB 패턴)
- Safe area 하단 padding 반드시 적용 (iPhone notch·홈바 대응)

### 아이콘

`material_icons` 패키지 기본 세트. 커스텀 아이콘은 최소화.

| 아이콘 | 의미 |
|---|---|
| `navigation` | 현재 위치 (방향 있음) |
| `place` | 목적지 마커 |
| `directions_walk` | 경로 진행 중 |
| `mic` | 음성 입력 |
| `search` | 목적지 검색 |
| `layers` | 층 전환 |
| `info_outline` | 건물 정보 Q&A |

## 6. 화면 명세

### 화면 목록

```
S1. 스플래시 / 권한 요청
S2. 야외 지도 (GPS 모드)
S3. 실내 전환 애니메이션
S4. 실내 지도 (PDR 모드)
S5. 목적지 입력 (검색 + 자연어)
S6. 경로 안내 (네비게이션 모드)
S7. 도착 / 완료
```

---

### S1. 스플래시 / 권한 요청

**목적**: 앱 로딩 + 위치·센서 권한 획득

**레이아웃**
- 중앙: 앱 아이콘 + 앱 이름 "Navigation"
- 하단: 권한 설명 텍스트 + "시작하기" CTA 버튼

**UX 규칙**
- 권한 거부 시 기능 제한 안내 (토스트). 강제 종료하지 않음.
- 로딩 상태는 LinearProgressIndicator (상단 얇은 바).

---

### S2. 야외 지도 (GPS 모드)

**목적**: 실외 이동 중 현재 위치 표시, 건물 입구 접근 유도

**레이아웃**
```
┌─────────────────────┐
│  [상단 바]           │  ← 반투명. 건물명 or "야외"
│                     │
│                     │
│    flutter_map      │  ← OSM 타일 지도. 전체 화면
│    (야외 OSM)        │
│      ◉ 현재위치      │
│                     │
│                     │
├─────────────────────┤
│  [하단 검색바]       │  ← "목적지를 입력하세요"  (터치 시 S5)
└─────────────────────┘
       [⊕ 내위치]      ← FAB, 우하단
```

**상세 규칙**
- 현재 위치 마커: 파란 원(GPS accuracy circle) + 방향 화살표
- GPS 정확도 낮을 때: 마커 색을 Warning(노랑)으로 변경 + "GPS 신호 약함" 배지
- 지도 타일: OpenStreetMap (API 키 불필요, 인터넷 연결 필요)
- 건물 입구 근처(반경 ~50m) 진입 시: "건물 감지 중..." 하단 스낵바

---

### S3. 실내 전환 애니메이션

**목적**: 실외 → 실내 전환 순간을 명확히 인지시킴

**레이아웃**
- 지도 위 전체 오버레이 (반투명 블랙)
- 중앙: 건물 아이콘 + "실내로 진입합니다" 텍스트
- 하단: 층 선택 (자동 감지 or 수동: "몇 층에 계신가요?")

**애니메이션 시퀀스**
1. OSM 지도 → 페이드아웃 (300ms)
2. 오버레이 등장 (200ms) + 텍스트 슬라이드업
3. 평면도 페이드인 (400ms)
4. 오버레이 사라짐 → S4로

**UX 규칙**
- 전환은 자동 (수동 버튼 없음). 심사위원에게 "자동"임을 보여주는 핵심 장면.
- 기압계 없는 기기: 층 수동 선택 UI 표시.

---

### S4. 실내 지도 (PDR 모드)

**목적**: 평면도 위에서 실시간 위치 추적

**레이아웃**
```
┌─────────────────────┐
│  [상단 바]           │  ← 건물명 + 현재 층 (예: "스타필드 3F")
│  [층 전환 버튼]      │  ← 우상단 (layers 아이콘 + 현재층 숫자)
│                     │
│    평면도 렌더링     │  ← flutter_map 커스텀 레이어
│    (GeoJSON 기반)   │
│      ◎ 현재위치      │  ← 보라 마커 (Indoor accent) + 불확실성 원
│                     │
│                     │
├─────────────────────┤
│  [하단 검색바]       │  ← "목적지를 입력하세요"  (터치 시 S5)
└─────────────────────┘
  [⊕ 내위치] [? 건물정보]  ← 우하단 FAB 두 개
```

**상세 규칙**
- 현재 위치 마커: 보라색 원 + 투명도 원(Particle Filter 분산 반영). 분산 클수록 원이 넓고 연함.
- 평면도: 벽(회색 선) + 복도(연한 회색 fill) + POI 아이콘(작게).
- PDR 갱신마다 마커 부드럽게 이동 (AnimatedContainer 또는 Tween).
- 층 전환: 층 버튼 탭 → 바텀시트에서 층 선택.
- "건물정보" FAB: RAG Q&A 패널 열기 (S6 아래 참고).

---

### S5. 목적지 입력

**목적**: 자연어 또는 POI 목록으로 목적지 선택

**레이아웃** (바텀시트 확장 패턴)
```
┌─────────────────────┐
│  ──── (드래그 핸들) │
│  [🎤  검색창        ]│  ← 마이크 아이콘 + 텍스트 입력
│  ─────────────────  │
│  최근 목적지         │
│  ├ 화장실 (2F)      │
│  ├ 스타벅스 (3F)    │
│  └ 편의점 (1F)      │
│  ─────────────────  │
│  이 건물의 POI 목록  │
│  ├ 📍 카페           │
│  ├ 🚻 화장실         │
│  └ 🏪 편의점         │
└─────────────────────┘
```

**자연어 입력 플로우**
1. 텍스트 입력 or 마이크 탭(STT)
2. 로딩 스피너 (RAG 처리 중, ~0.5~2초)
3. 결과 1개: 바로 경로 시작 → S6
4. 결과 여러 개: 후보 리스트 표시 → 선택 → S6
5. 결과 없음: "찾을 수 없어요. 다시 입력해볼까요?" 빈 상태

---

### S6. 경로 안내 (네비게이션 모드)

**목적**: 목적지까지 실시간 경로 안내

**레이아웃**
```
┌─────────────────────┐
│  [상단 안내 카드]    │  ← 다음 방향 (예: "10m 직진 후 우회전")
│                     │
│    평면도            │
│    경로 오버레이 ───┤  ← Primary(파랑) 선. 지나간 구간은 연하게
│      ◎ 현재위치     │
│      📍 목적지      │
│                     │
├─────────────────────┤
│  [하단 ETA 카드]    │  ← "목적지까지 약 2분 / 150m"
│  [경로 취소]        │  ← 작은 텍스트 버튼
└─────────────────────┘
```

**RAG Q&A 패널** (건물정보 FAB 탭 시)
- 바텀시트 슬라이드업
- 채팅 버블 UI (사용자 입력 / AI 답변 교대)
- 예시: "화장실 몇 시까지 이용 가능해요?" → "본관 화장실은 22시까지 운영합니다."

**경로 업데이트 규칙**
- PDR 위치 갱신마다 경로 re-render.
- 경로 이탈 감지 (예: 1.5m 이상 이탈) → 자동 경로 재계산 + "경로를 재탐색합니다" 스낵바.
- 층 변경(계단·엘리베이터) 시: 안내 카드에 "3층으로 올라가세요" 표시 + 층 자동 전환.

---

### S7. 도착 / 완료

**목적**: 목적지 도달 알림

**레이아웃**
- 전체화면 성공 오버레이 (1~2초 후 자동 닫힘)
- 중앙: 체크 아이콘 (Success 초록) + "도착했습니다!"
- 하단: "새 목적지 탐색" 버튼

| `StatusBadge` | GPS 약함 / 위치 정확도 상태 배지 |

## 8. 디자인 시 주의사항

- **평면도 GeoJSON은 flutter_map 커스텀 레이어**로 렌더링. 지도 라이브러리(google_maps 등) 아님.
- 실내 좌표계는 **건물 로컬 좌표(미터)** 기반. 위경도가 아니므로 핀 위치 변환 로직 필요.
- 네트워크 없이도 PDR은 동작해야 함. 온디바이스 기능은 오프라인 상태에서도 UI가 정상 동작.
- 백엔드(FastAPI) 응답 지연 시 로딩 상태 반드시 표시 (평면도 로딩, RAG 검색).
- 경진대회 데모 환경: 한 건물, 한 층 or 두 층이 최소 목표. 다건물 UI는 1차 범위 외.

## 9. UI 디자인 작업 프롬프트 (Claude Design용)

아래 블록을 Claude Design에게 전달해 특정 화면 디자인을 요청할 때 사용한다.
`요청할 화면`, `추가 조건` 두 줄만 바꿔서 쓰면 된다.

---

```
design.md 파일을 읽고, 거기 담긴 서비스 컨텍스트·디자인 원칙·화면 명세·디자인 시스템을 기반으로 아래 화면을 디자인해줘.

요청할 화면: [S1~S7 중 선택, 예: S4 실내 지도]
추가 조건: [선택 — 예: "위치 불확실성 높음 상태" / "RAG 응답 대기 중 상태" / 없음]

---

출력 규칙 (항상 적용):
- 와이어프레임이나 스케치 스타일 금지. 실제 출시 앱 수준의 고해상도 UI로 만들어줘.
- 평면도는 실제 도면처럼: 벽은 진한 선, 복도는 연한 배경 fill, POI는 아이콘+라벨
- 현재 위치 마커는 #6C3FE0 보라색 실제 마커, 불확실성 원은 반투명 보라
- 상단바·검색바·FAB는 Material 3 스타일로 그림자·elevation 표현
- 손글씨 폰트 금지. 시스템 산세리프체 사용
- 포스트잇·어노테이션 없이 화면만 출력
```
