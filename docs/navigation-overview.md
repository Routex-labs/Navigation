# Navigation 프로젝트 개요

## 목적

Navigation은 경진대회 제출을 목표로 하는 경로 안내/이동 보조 프로젝트다.
초기 문서는 구현 전 의사결정과 역할 분담을 빠르게 고정하기 위한 뼈대이며, 실제 기능이 확정될 때마다 갱신한다.

## 현재 결정

| 항목 | 결정 | 메모 |
|---|---|---|
| 프로젝트명 | Navigation | 저장소, Docker 이미지 태그, 문서 제목에 동일하게 사용 |
| 목표 형태 | 데모 가능한 서비스 | 발표에서 바로 보여줄 수 있는 얇은 수직 슬라이스 우선 |
| 핵심 컴포넌트 | client / api / routing / data / demo / infra | 상세 스택은 VERSION.md에서 관리 |
| 운영 방식 | GitHub Flow + Projects | CI/CD 워크플로는 `.github/workflows/`에서 관리 |

## 아직 정해야 할 것

- 실제 사용자 시나리오: 누구의 어떤 이동 문제를 해결하는지
- 입력 데이터: 지도 API, 정적 데이터, 센서/위치 데이터, 직접 제작 데이터 중 무엇을 쓸지
- 경로 산출 방식: 단순 최단 경로, 조건 기반 추천, 위험/혼잡 회피 등
- 데모 화면: 지도 중심인지, 경로 비교 중심인지, 대시보드 중심인지
- 평가 지표: 정확도, 이동 시간 절감, 안전성, 사용성, 발표 임팩트 중 무엇을 우선할지

## 문서 갱신 규칙

- 결정된 내용은 표에 추가하고, 바뀐 결정은 삭제하지 말고 `HISTORY.md`에 변경 이유를 남긴다.
- 조사 근거와 링크는 [research-notes.md](research-notes.md)에 모은다.
- 아직 구현 방향이 확정되지 않았으므로 별도 로드맵 문서는 만들지 않는다.
- 일정과 구현 순서가 정해지면 이 문서에서 먼저 합의한 뒤 필요한 파일을 새로 만든다.
